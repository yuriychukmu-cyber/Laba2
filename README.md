# Лабораторная работа №2
## Тема: «С++. Создание контейнеров. Исключения. Шаблоны»

## 1. Цель работы
Изучить принципы разработки пользовательских контейнеров, наследования, обработки исключений и шаблонов классов на C++17. Реализовать тестирующее графическое приложение на Qt Widgets.

## 2. Краткое описание части 1
В первой части реализована очередь `BaseQueue`, которая хранит элементы двух типов: число и строка.

- Элемент представлен классом `QueueItem` с полем типа (`Number` или `String`) и соответствующим значением.
- Базовая очередь поддерживает операции:
  - добавление числа (`pushNumber`)
  - добавление строки (`pushString`)
  - удаление элемента по индексу (`removeAt`)
  - удаление первого элемента (`pop`)
  - вывод содержимого (`toString`)
  - проверка на пустоту и получение размера.
- От `BaseQueue` наследуется `SortableNumberQueue`, добавляющий сортировку числовых элементов (`sortNumbersOnly`).

## 3. Краткое описание части 2
Во второй части очередь преобразована в шаблонный класс `TemplateQueue<T>`.

Поддержаны операции:
- `push`
- `pop`
- `front`
- `isEmpty`
- `size`
- `toString`

В GUI для демонстрации используются две специализации:
- `TemplateQueue<int>`
- `TemplateQueue<QString>`

## 4. Классы в проекте
### Часть 1
- `QueueItem` — элемент очереди (число/строка)
- `QueueException` — базовое исключение части 1
- `EmptyQueueException` — исключение при удалении из пустой очереди
- `InvalidInputException` — исключение некорректного ввода
- `BaseQueue` — базовая очередь
- `SortableNumberQueue` — очередь с сортировкой чисел

### Часть 2
- `TemplateQueueEmptyException` — исключение пустой шаблонной очереди
- `TemplateQueue<T>` — шаблонная очередь

### GUI
- `MainWindow` — главное окно с блоками для тестирования обеих частей

## 5. Реализованные исключения
1. `EmptyQueueException` — попытка удаления элемента из пустой очереди (часть 1)
2. `InvalidInputException` — пустой ввод или неверный формат данных (часть 1, используется в GUI)
3. `IndexOutOfRangeException` — индекс удаления вне диапазона в части 1
4. `TemplateQueueEmptyException` — попытка удаления/чтения из пустой шаблонной очереди (часть 2)
5. `TemplateQueueIndexOutOfRangeException` — индекс удаления вне диапазона в части 2

## 6. Логика сортировки чисел в очереди
Метод `sortNumbersOnly()` работает по простому и наглядному алгоритму:
1. Из очереди извлекаются только числовые значения.
2. Эти числа сортируются по возрастанию.
3. Далее очередь проходится повторно: на местах, где стояли числовые элементы, подставляются отсортированные числа.
4. Строковые элементы не перемещаются и остаются на своих исходных позициях.

После добавления/удаления/сортировки вывод в `QTextEdit` не обновляется автоматически. Актуальное состояние отображается только по кнопке **Показать очередь**.

Пример:
- До сортировки: `[10, "abc", 3, "q", 7]`
- После сортировки: `[3, "abc", 7, "q", 10]`

## 7. Сборка проекта в Qt Creator
1. Откройте Qt Creator.
2. Выберите **Open File or Project...** и откройте `CMakeLists.txt`.
3. Выберите доступный набор (Kit) с Qt5 или Qt6.
4. Нажмите **Configure Project**.
5. Соберите проект (**Build**) и запустите (**Run**).

## 8. Как протестировать программу
### Блок «Часть 1»
1. Выберите тип (`Число` или `Строка`) и введите значение.
2. Нажмите **Добавить**.
3. Нажмите **Показать очередь** для просмотра состояния.
4. Добавьте смешанные данные и нажмите **Сортировать числа**.
5. Введите индекс и нажмите **Удалить по индексу**.
6. Попробуйте удалить из пустой очереди, чтобы увидеть обработку исключения.

### Блок «Часть 2»
1. Выберите тип шаблонной очереди (`int` или `QString`).
2. Введите значение и нажмите **Добавить**.
3. Нажмите **Показать очередь**.
4. Введите индекс и нажмите **Удалить по индексу**.
5. Попробуйте некорректный ввод для `int` или удаление из пустой очереди.

## 9. Текстовое описание диаграммы классов
- `BaseQueue` содержит контейнер `std::deque<QueueItem>` и базовые операции очереди.
- `SortableNumberQueue` наследует `BaseQueue` и добавляет `sortNumbersOnly()`.
- `QueueItem` агрегируется внутри `BaseQueue` как тип элемента.
- `QueueException` является базовым классом для `EmptyQueueException` и `InvalidInputException`.
- `TemplateQueue<T>` — отдельная обобщённая очередь с собственным исключением `TemplateQueueEmptyException`.
- `MainWindow` использует (`has-a`) объекты `SortableNumberQueue`, `TemplateQueue<int>` и `TemplateQueue<QString>` для GUI-тестирования.
